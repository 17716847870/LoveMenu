# LoveMenu 后台首页（Dashboard）设计任务文档

---

# 一、任务目标

你需要完成 LoveMenu 后台首页（Dashboard）的设计与开发。

该页面用于展示系统核心数据，并通过图表进行可视化表达，强调情侣互动数据和趣味性。

---

## 核心要求

1. 必须区分 PC 端 和 移动端（不可简单缩放）
2. PC 端：数据分析导向（高信息密度，便于管理）
3. 移动端：轻量浏览导向（可快速查看互动情况）
4. 必须使用图表或可视化组件展示数据
5. UI 必须符合情侣主题（温柔、互动、可爱）

---

# 二、数据模块设计

---

## 1. 核心指标

- 今日互动订单数（情侣间新提交的订单）
- 今日亲亲 + 贴贴总数（订单积分总计）
- 紧急想吃订单数（标记优先处理）
- 新增反馈数（用户提交意见）
- 新增想吃清单数（用户提交未上菜单食物）

---

## 2. 趋势数据

- 近7天互动订单趋势
- 近7天亲亲 + 贴贴积分趋势
- 近7天紧急想吃趋势

---

## 3. 分布数据

- 菜品分类占比（情侣点餐偏好）
- 订单来源占比（普通点餐 / 紧急想吃 / 想吃清单转菜单）

---

## 4. 热门数据

- 热门菜品 TOP5（按点赞数 + 点餐次数）
- 热门提交想吃食物 TOP5（未上菜单但被多情侣提及）

---

# 三、PC端 Dashboard 设计

---

## 页面结构

Header

数据卡片（横向 4~5 列）

趋势图（折线图）

分布图（左右布局）

热门菜品 & 热门想吃清单（表格）

---

## 1. 数据卡片（Stat Cards）

布局：

- 横向排列
- 4~5 列

内容示例：

- 今日互动订单数
- 今日亲亲 + 贴贴总数
- 紧急想吃订单数
- 新增反馈数
- 新增想吃清单数

设计要求：

- 圆角卡片
- 粉色主题高亮
- 大数字显示
- 支持趋势变化（箭头 ↑ ↓）
- 卡片图标可爱、情侣化

---

## 2. 趋势图（折线图）

内容：

- 近7天互动订单趋势
- 近7天亲亲 + 贴贴积分趋势
- 近7天紧急想吃趋势

设计要求：

- 双折线图或多折线图
- Hover 显示具体数值
- 可切换不同趋势指标
- 配色温柔、可爱

---

## 3. 分布图（饼图 / 圆环图）

左侧：

- 菜品分类占比（情侣点餐偏好）

右侧：

- 订单来源占比（普通 / 紧急 / 想吃清单）

设计要求：

- 饼图或圆环图
- 带图例
- Hover 显示具体数值
- 配色柔和、符合情侣主题

---

## 4. 热门菜品 & 想吃清单表格

表格字段：

- 菜名 / 想吃名称
- 分类
- 点餐次数 / 被提次数
- 点赞数
- 状态（已上菜单 / 待审核）

设计要求：

- 表格形式
- 支持排序
- 高信息密度，方便后台管理
- 行动按钮：编辑 / 删除 / 上架 / 审核通过

# 六、API 设计

---

## 1. 获取统计数据

GET /api/dashboard/stats

返回：

- interactionsToday: number
- pointsToday: number（亲亲 + 贴贴总数）
- priorityOrders: number
- newFeedback: number
- newWishlist: number

---

## 2. 获取趋势数据

GET /api/dashboard/trends

返回：

- last7DaysInteractions: array
- last7DaysPoints: array
- last7DaysPriorityOrders: array

---

## 3. 获取分布数据

GET /api/dashboard/distribution

返回：

- categoryDistribution: array
- sourceDistribution: array（normal / emergency / wishlist）

---

## 4. 获取热门数据

GET /api/dashboard/top-items

返回：

- topDishes: array（按点餐次数 + 点赞数）
- topWishlist: array（按提交次数）
